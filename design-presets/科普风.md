# 科普风 / Science-Explainer · 叙事调性档

> 拷成项目根 `design.md` 使用。只写节奏 / 语气 / 观众 / 风格档，**不写色值 / 字体 / 光效**。
> 视觉真相在 `src/methods/designs.ts` tokens，动效在 `docs/MOTION.md`，选法在 `docs/DIRECTION.md`。

## 设计预设（视觉血统）

- 预设 id：`nocturne`（克制深色，冷静权威、数据向）
  - 想要更冷更精确的黑白气场可切 `swiss`。
- 数据段与叙述段可在片内换档，但一段内只用一个预设。

## 风格档案（DIRECTION §二）

- 主档：**A 极客湾式**数据评测——结论先行，每段以一个数据事件开场。
- 纯概念铺陈 / 章节框架可借 **C 印刷工坊经典**。

## 节奏密度

- 目标：每 **2.5–3.5s** 一个视觉事件；数据变化本身即事件。>6s 无事件即违规。
- tempo：`snappy`（信息不是戏剧，干脆利落），章节框架处回到 `deliberate`。

## 语气 / 目标观众

- 观众：想弄懂原理、看得进数据的中文观众。
- 语气：冷静、准确、克制；先给结论再展开；一屏一焦点，别堆信息。

## 方法偏好（语义，落到 DIRECTION §一 映射表）

- 单个核心数值 → `hf-mega-counter`；多主体排名/份额 → `rm-d3-bar-race`。
- 趋势/随时间变化 → `rm-d3-line-draw`；A vs B 参数拉表 → `hf-versus-panel`。
- 关键事实强调 → `hf-scribble-annotate` 圈注 或 spotlight 调光（MOTION §四）。
- 讲到某数据时其余降 35% 透明度（spotlight），别让画面同时大动。

## 配音

- 默认解说：`presenter_male`（沉稳播客感）。
- 个人经历/自述插叙可切 `audiobook_male_1`；「另一派观点」用 `presenter_female`。
- 每 60s 约 2–3 次换声。

## Don'ts（继承印刷工坊命门）

- 图表镜禁戏剧化 ease、禁 kenburns 乱推拉（数据要「稳」）。
- 禁渐变 / 发光 / 投影 / AI 金光紫气（MOTION.md §一红线）。
- 动效参数看 `docs/MOTION.md`，色值/字体看 design tokens——都不在这里写。
