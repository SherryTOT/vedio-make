# MOTION.md — 动效法典(参数级,所有模型照此执行)

> 目标:把「好动效」从模型的临场判断变成可照抄的规范。对标:极客湾(数据动效)、小Lin说(拼贴叙事动效)。
> 地位:**动效唯一事实源**。与代码冲突时以本文件为准并改代码;与 design.md 旧 Motion 段冲突时本文件胜(旧段已废,见 DIRECTION.md)。
> 审美底线继承印刷工坊:动效管「节奏与运动」,不许引入渐变/发光/投影/粒子——**克制的画面 + 充沛的节奏**,两者不矛盾。

## 一、红线(一票否决,lint 级)

1. 禁多次回弹:spring damping < 10、bounce 弹 2 次以上。过冲最多 1 次、幅度 ≤8%。
2. 禁大角度 3D:rotationX/Y 超过 ±6° 的翻转卡废除(纸不做 3D 翻转)。旧 design.md 的 ±25° 相机规则作废。
3. 禁 blur/glow/发光脉冲/粒子星光/彩虹 stagger。转场不用 dissolve 糊脸,硬切为主。
4. 禁 emoji 上画面;图形一律矢量图标或生图素材。
5. 数字必须 `font-variant-numeric: tabular-nums` 或等宽字体,滚动不许抖宽。
6. 大标题禁两端对齐;中文行长 ≤18 字;安全区:左右 ≥120px,上下 ≥80px(1080p 基准)。
7. 色彩只准来自 `ctx.design.*` tokens,任何写死 hex 即违规(含生图 prompt 里的色调词)。

## 二、节奏学(片子的心跳)

- **视觉事件密度**:解说片每 **2.5–4s** 必须有一个视觉事件(新元素进场/强调/数据变化/镜头微动改变)。**>6s 无事件 = 违规**(纳入 slideshow 单调检测)。
- **与解说同步**:动效起点对齐台词时间戳,**提前 0.1–0.2s 起手**(anticipation);关键词的强调动效必须落在该词发音区间内。
- **进出场时长表**:

| 元素 | 进场 | 退场(=进场×0.7,in-cubic) |
|---|---|---|
| 小元素(标注/关键词/图标) | 0.3–0.45s | 0.2–0.3s |
| 卡片/图表单元素 | 0.6–0.9s | 0.4–0.6s |
| 全屏布局/场景 build | 0.9–1.2s | 0.6–0.8s |

- **easing 选择表**:进场 `power3.out`/`power4.out`(out-cubic/quart);位移换位 `power2.inOut`;强调 spring(damping 12–16);常驻循环 `sine.inOut`。进场绝不用 in-out(拖泥带水)。
- **stagger 公式**:兄弟元素 60–120ms;列表 >6 项降到 40–80ms;整组 build 总时长 ≤1.5s。
- **一屏一焦点**:同屏同时动的东西 ≤1 主 + 1 辅;文字与图表不同时大动。
- **镜头微动层**(图表/卡片场景通用):整场 scale 1.00→1.02 匀速缓推 8–10s,给画面「活着」的呼吸,幅度绝不超 1.03。
- **转场纪律**:默认硬切;每 3–5 镜允许一次「纸感横擦」(纯色块 0.5s power2.inOut 扫过);全片转场种类 ≤2 种。

## 三、动效词汇表 · 新方法包(7 个,P1 实装)

每条 = 何时用 / 进场 / 常驻 / 退场 / 关键参数。实现引擎按 methods/catalog.json 决策矩阵。

### 数据系(极客湾式)
1. **`rm-d3-bar-race` 柱状竞速** — 多主体排名对比。换位:y 位移 0.8s power2.inOut + 数值同步滚动;新第一名条染 `accent`、旧的回 `accent2`;每次重排间隔 ≥1.2s。数据 ≤8 条。
2. **`rm-d3-line-draw` 曲线扫描** — 趋势叙事(功耗曲线式)。stroke-dashoffset 画线 1.2–2s power4.out + 游标点沿线;拐点标注在游标到达时 pop(scale 0→1 back.out(1.4) 0.4s);多曲线 stagger 0.3s,当前讲的线全彩、其余 35% 透明度。
3. **`hf-mega-counter` 大数字大屏** — 单个核心数据(跑分/价格/百分比)。数字 0→N 滚动 1.2s power4.out;单位小字延后 0.3s 淡入;对比箭头(用 design ok/alert 色)最后 pop;数字字号 ≥屏高 22%。
4. **`hf-versus-panel` 对比面板** — A vs B(可三列)。两侧对向滑入 0.6s stagger 0.1s;参数行「讲到才点亮」(逐行 0.35s,行间隔跟台词);优势行文字染 accent + scale 1.04 spring;劣方整卡 opacity 0.85。

### 叙事系(小Lin说式)
5. **`hf-sticker-pop` 贴纸上台** — 人物/公司/产品实体登场。素材=生图白底单体经 matte 抠像。进场 scale 0.6→1 back.out(1.4) 0.5s + 旋转 ±8°→±2° 落定;常驻微摆 rotate ±1.5°、4–6s sine.inOut 循环;同屏贴纸 ≤4 张;出场缩小+淡出 0.3s。
6. **`hf-scribble-annotate` 手绘圈注/箭头** — 强调与指向。SVG 路径描边逐笔 0.6s(路径带手抖噪声,2px 墨色或 accent 笔触);箭头沿贝塞尔画出 0.8s,可带小图形沿线流动循环(流速 1.2s/程);一镜内手绘元素 ≤3 笔。
7. **`hf-word-punch` 关键词弹字** — 台词金句/关键词。底色块 scaleX 0→1(0.25s power3.out)先行,字随后 scale 1.3→1(0.35s)落下;严格对齐该词发音时刻;每 15s 至多 2 次,滥用即廉价。

### 既有 16 方法的动效校准(P1 顺手)
- `hf-stat-counter`:升级为 mega-counter 的简版,补 tabular-nums 与 power4.out。
- `rm-d3-bar-chart`:现有 stagger 6 帧/spring damping 12 保留;数值文字改等宽;加「讲到某柱时其余降 35% 透明度」的 spotlight 参数位。
- `rm-image-kenburns`:推拉固定匀速 1.03–1.08,禁 in-out 呼吸。
- `hf-css-fade`:仍是兜底,但连续 2 镜兜底触发单调警告(slideshow.ts 加维度)。

## 四、通用强调技法(所有方法可组合)

- **Spotlight 调光**:讲到谁,其余元素 opacity→0.25(0.4s),主体 scale 1.05;收回 0.3s。
- **数据强调**:变色(→accent)+ 上浮 6px + spring 落回,三件套一起 0.5s 内完成。
- **定帧收束**:片尾金句定帧 ≥1.5s,只留微呼吸(scale ±0.5%),不加任何新动效。

## 五、好坏对照(执行模型最易犯的错)

| ❌ 常见坏 | ✅ 本法典 |
|---|---|
| 每个元素都在动、满屏热闹 | 一屏一焦点,其余静止或调暗 |
| 进场用 1.5s+ 的 in-out 慢飘 | 0.3–0.9s power3/4.out,干脆 |
| 强调=放大加发光 | 变色+上浮+spring 落回,无光效 |
| 转场轮换 5 种花样 | 硬切为主,全片 ≤2 种 |
| bounce 弹三下卖萌 | 过冲 1 次 ≤8% |
| 数字变化跳帧/抖宽 | tabular-nums + 滚动插值 |
| 动效与台词各动各的 | 起点对齐词时间戳,提前 0.1–0.2s |
